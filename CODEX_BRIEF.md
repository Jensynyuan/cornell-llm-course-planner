# Codex Brief — Cornell LL.M. Planner v5.2

## 产品目标

维护一个中文优先、离线可用的美国法学院 LL.M. 选课规划器。Cornell 是默认且完整预装的数据集；其他学校通过适配器导入。

## 不可破坏的规则

1. Cornell 数据必须在构建阶段完成抓取与机器翻译，最终用户不得被要求配置 OpenAI API。
2. `data/cornell.catalog.zh-CN.js` 必须可以离线直接加载；不要把启动流程改成联网拉取。
3. 课程目录与学期班次要分开：没有班次时间的课程可搜索，但不可伪造课表色块。
4. 每门课程必须显示一个 NY Bar 状态和一个选课权限状态。
5. 所有标签使用统一高度、字号、圆角和内边距。
6. 校区不是教室。只有 meeting-level 楼宇／房间字段才能显示为具体地点。
7. IALS 仅在 Orientation 实际日期内出现。
8. 拖拽课程卡后，按照所选班次官方日期和时间自动落位；时间重叠必须弹出明确冲突提示。
9. 课程推荐不得写死某个用户的履历；使用职业方向和课程偏好的组合式评分。
10. 外校导入的 NY Bar 状态默认 `review`，除非有该校官方依据。
11. Cornell 用户可见中文字段必须在构建阶段完成；`titleEn`、`descriptionEn`、英文限制和英文先修字段不得进入 `data/cornell.catalog.zh-CN.js`，不得在前端作为回退内容展示。
12. 每次发布必须运行 `node scripts/verify_release_v5_2.mjs`；课程数、中文字段、重复课程号、v5.2 术语检查和浏览器目录一致性均须通过。

## Cornell 更新工作流

1. 从 Cornell 官方 Catalog 和 Class Roster/API 获取原始数据。
2. 合并课程目录与当前学期 section/meeting 元数据。
3. 在构建机器上完成批量机器翻译并应用锁定法律术语词表；不要在浏览器端翻译。
4. 输出：
   - `data/cornell-law-YYYY-YY.zh-CN.json`
   - `data/cornell.catalog.zh-CN.js`
5. 执行完整性检查：课程数、中文字段非空、班次日期、时间格式、重复 ID、NY Bar 标签、英文回退字段和锁定术语检查。
6. 启动本地服务器，以 1600×1000 视口截取课程检索、课程详情和课表页面。
7. 通过后再生成 ZIP。

## 其他法学院适配

按照 `T14_IMPORT_GUIDE.md` 和 `IMPORT_SCHEMA.md` 创建独立数据文件。不要假定 T14 存在统一 API，也不要绕过学校权限。

## 交互与教室规则

- 课程卡必须支持原生鼠标拖拽到左侧“我的课表”、已选课程区和周课表；不要只支持点击按钮。
- 多班次先弹出班次选择，冲突必须弹窗，不得静默覆盖。
- 教室同步使用 `/api/cornell/sync-rooms`，按班号从 Fall 2026 官方课表读取；不得用校区或院系名称代替房间。
- 用户页面不得出现“对开发者”“读取用户简历”“构建阶段”等内部实现式文案。
- 保留主页面署名：本页面由袁敬轩使用 Codex 设计。

## v5.2 翻译发布规则

- 使用 `scripts/build_v5_2_release.mjs` 将已批准的中文源数据构建为 JSON 和浏览器目录；不要手工修改两份运行数据中的单门课程。
- `data/qa/` 中的三份 JSON 是本次发布的术语及翻译质检记录；后续替换课程数据库时须同时产生新的检查记录。
- 允许保留课程号、教师姓名、缩写、班号及官方链接；英文课程正文只应在官方页面中查看，不应出现在本地中文详情中。
